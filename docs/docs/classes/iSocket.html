    iSocket Class

    提供与应用服务器、第三方服务器等远端进行WebSocket交互的服务类。

    1. 主框架已加载，无需再次引入。
    2. 此类为服务，需要依赖注入才能使用。
     JS:
 app.controller('***Controller', ['iSocket', function(iSocket) {
     // TODO
 }]);

    close

    close ()

    Defined in temp\iiw.core\webapp\js\services\server\socket.js:200

    主动关闭与应用服务器的WebSocket连接。

    Example:

     // 注入iSocket服务。
 app.controller('***Controller', ['iSocket', function(iSocket) {
     // 关闭WebSocket。
     iSocket.close();
 }]);

    connect

    connect ()

    Defined in temp\iiw.core\webapp\js\services\server\socket.js:170

    与服务器建立WebSocket的连接，仅支持与应用服务器连接，不支持与第三方地址的WebSocket连接。

    • WebSocket的建立时机由模块开发者把握，原则上建议在登陆后自动建立连接。
    • 在调用本方法后，无论时候连接上，或连接上后断链，都会自动尝试重连。

    Example:

     // 注入iSocket服务。
 app.controller('***Controller', ['iSocket', function(iSocket) {
     // 连接应用服务器WebSocket。
     iSocket.connect();
 }]);

    Events

    on

    Defined in temp\iiw.core\webapp\js\services\server\socket.js:219

    监听应用服务器主动发布的WebSocket消息。

    • 注意，多次调用该方法监听同一个事件会多次触发，并且由于闭包的原因，无法从内存中删除，建议开发者在模块框架下使用服务方式进行监听，只监听一次，避免资源的浪费。

    Parameters:

    nametypeflagdescription
    eventName String

    监听的消息名称

    callback Function

    回调函数，监听后，服务器发送对应的消息时，调用回调函数方法。

    Example:

     // 注入iSocket服务。
 app.controller('***Controller', ['iSocket', function(iSocket) {
     // 监听testEvent消息。
     iSocket.on('testEvent', function(data) {
         // TODO
         console.log(data); // 应用服务器基于WebSocket向客户端发送了一个testEvent消息，消息内容为data。
     });
 }]);
